`.magics` = ZIP-архив с **подменённой сигнатурой**: байты `MT` (0x4D54) вместо `PK` (0x504B). Открыть: заменить во всём файле `MT\x03\x04`→`PK\x03\x04`, `MT\x01\x02`→`PK\x01\x02`, `MT\x05\x06`→`PK\x05\x06` (а также `\x07\x08`, `\x06\x06`, `\x06\x07`), затем `zipfile.ZipFile(io.BytesIO(fixed))`.

Внутри:
- `Stl{uuid}_vertices` — вершины, **int32 xyz в единицах 0.1 мкм** → умножить на **0.0001** для мм.
- `Stl{uuid}_surfaces` — треугольники, **uint32** индексы (по 3 на грань).
- `SupportSurfaces_ox...` — геометрия поддержек. **Формат НЕ декодируется тривиально** (пробовал 2026-06-16: после int32-граней идёт не чистый float32-массив вершин, а перемешанная структура — служебные поля/интерливинг/нормали + нулевой падинг). Расковыривать дальше — рабблхол. Брать поддержки НЕ отсюда, а из STL-файлов на диске (`s_Деталь_N.stl`).
- `header.xml` — **зашифрован** (имена деталей, трансформы, метаданные напрямую НЕ читаются).

Каждый `Stl{uuid}` — это уже размещённый на платформе инстанс (с применённым трансформом). Объём = `abs(trimesh.Trimesh(v,f,process=False).volume)`. Группировка инстансов по (n_verts, n_faces, volume) даёт число копий каждой детали. Изредка встречается `_vertices` без пары `_surfaces` — пропускать.

Валидировано (2026-06-15): объёмы совпали с известными STL до сотых (Hvostovik 27.9, Korpus 10.4, Box 2.6 см³), высоты сборки совпали со слоями логов (04.06: magics 119 мм = 1983×0.06).

**ВАЖНО — на практике magics обычно НЕ нужен:** в папке печати уже лежат готовые STL — `Деталь_N.stl` (по копии, с реальными XY) и `s_Деталь_N.stl` (поддержки). Это и есть полная размещённая платформа. Декод тел из magics к тому же **недосчитывает** (пропускает инстансы без пары `_surfaces` — 04.06 дал ×4 занижение). Поэтому: **бери STL с диска**, magics-декод — только если STL нет. Подробности калибровки → [[m450-print-time-project]].
